The Inspectoor

Deterministic extraction and exploration of all Ethereum specification data.

The Inspectoor parses every Ethereum spec repo and builds a structured index of types, endpoints, constants, presets, and containers. It tracks changes across forks, diffs fields between versions, and makes everything searchable through a browser UI and an MCP server for AI agents.

Indexed Specifications

consensus-specs execution-specs execution-apis beacon-apis builder-specs relay-specs remote-signing-api

Explorer UI

The web interface has six main views:

Specs
Browse all indexed spec repos. See which forks are available and jump into any spec's types or endpoints.
Types
Search and inspect every container, enum, and type alias across all specs. View fields, see which fork introduced a type, and trace its dependencies.
Endpoints
Browse all API endpoints from beacon-apis, execution-apis, builder-specs, and more. See parameters, request/response schemas, and which spec version defines them.
Diff
Compare types between any two forks. See exactly which fields were added, removed, or changed. Useful for tracking breaking changes across upgrades.
PRs
Index open pull requests against spec repos. See what types and endpoints a PR would change before it merges.
Visualizer
Interactive graph of type dependencies. See how containers reference each other and trace the full dependency tree for any type.

MCP Server

The Inspectoor includes a Model Context Protocol (MCP) server so AI agents (Claude, Cursor, Windsurf, etc.) can query Ethereum spec data programmatically. Connect it to any MCP-compatible client and your agent gets direct access to the full spec index.

Available Tools

lookup_type
Get full definition of a type by name and fork
diff_type
Compare a type between two forks
what_changed
List all type changes between two forks
search
Full-text search across all specs and types
trace_type
Get the full dependency tree of a type
lookup_endpoint
Get details of an API endpoint by path
list_specs
List all indexed spec repos and their forks
list_prs
List open PRs indexed against spec repos
index_pr
Index a specific PR to see what it changes

Setup

Requires Python 3.10+ and git. One command builds everything.

git clone https://github.com/JasonVranek/inspectoor
cd inspectoor

# Install dependencies
pip install pyyaml mcp

# Build everything: clones all 7 spec repos, extracts, links, builds catalog
python3 build.py --all

# Open the explorer
open docs/index.html

That's it. build.py --all clones all spec repos to ./repos/specs/, builds per-spec indexes, runs cross-reference linking, and assembles the catalog. First run takes a few minutes to clone. Subsequent runs pull updates and rebuild.

MCP Server

Start the MCP server for AI agent integration:

# Start the server (stdio transport)
python3 server.py --catalog docs/catalog.json

For MCP client configuration (Claude Desktop, Hermes, Cursor), add to your config:

mcp:
  inspectoor:
    command: "uv"
    args:
      - "run"
      - "--with"
      - "mcp"
      - "--with"
      - "pyyaml"
      - "python3"
      - "/path/to/inspectoor/server.py"
      - "--catalog"
      - "/path/to/inspectoor/docs/catalog.json"

Including PR Data

To track open pull requests against spec repos (requires a GITHUB_TOKEN):

GITHUB_TOKEN=your_token python3 build.py --all --include-prs
